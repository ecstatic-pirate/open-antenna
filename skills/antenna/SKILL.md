# Antenna — Daily Intelligence Briefing

Orchestrator skill. Scans configured sources, processes content via sub-skills, assembles digest, optionally converts to PDF.

---

## Setup

The config file path is passed as an argument. Read it first:

```bash
cat {config_path}
```

All behavior — channels, HN settings, output dir, profile — comes from config. Never hardcode values.

---

## Step 1: Scan Sources

Run both scanners. Scanners live alongside this package in `scanners/`.

### YouTube

```bash
python3 {scanners_dir}/youtube_scanner.py --config {config_path}
```

Output: JSON array of `{title, url, channel, published, duration_seconds}` objects.

If the scanner errors, log a warning and continue with HN.

### Hacker News

```bash
python3 {scanners_dir}/hn_scanner.py --config {config_path}
```

Output: JSON with `{story_count, stories}` where each story has `{title, url, hn_url, score, comment_count, author, created_date}`.

If the scanner errors, log a warning and continue with YouTube.

---

## Step 2: Process YouTube Videos

For each video found:

1. Invoke `/read-podcast {url}` — but skip board creation and output file. Extract content only and return a structured summary.
2. If no transcript is available, skip this video entirely. Note it in the digest footer.
3. Collect: title, channel, published date, duration, big picture (1 sentence), key ideas (bullet list).

If no videos found: note "No new long-form content from tracked channels today." in the digest.

---

## Step 3: Process Hacker News

For each story in the top N (configured in config.yaml):

1. Fetch the article via Jina Reader:
   ```bash
   curl -s "https://r.jina.ai/{url}"
   ```
   If Jina fails, try WebFetch. If both fail, skip silently.

2. Assess relevance using the user's profile from config:
   - Name: `{profile.name}`
   - Role: `{profile.role}`
   - Topics: `{profile.topics}`

   If the article is not useful for someone with this profile, skip it. Irrelevant articles do not appear in the digest.

3. For relevant articles, write a scannable summary:
   - **Bold 1-sentence big picture first.** What the article actually says.
   - Bullet points for key claims. Each bullet = one concrete claim, fact, or insight.
   - Short sentences. Plain language. No jargon.
   - Keep specific numbers, names, examples — those make it memorable.
   - Flat list preferred. Sub-bullets only for enumerated steps.
   - If the article is paywalled or fails to load, skip it.

---

## Step 4: Write the Digest

Get today's date:
```bash
date +%Y-%m-%d
date +'%A, %B %-d'
```

Write to `{output_dir}/YYYY-MM-DD-digest.md`:

```markdown
---
type: antenna-digest
date: YYYY-MM-DD
sources: youtube, hackernews
youtube_videos: N
hn_articles: N
profile_name: {profile.name}
---

# Daily Intelligence Briefing — {DayName}, {Month} {Day}

---

## YouTube

### [{Video Title}]({url})
**Channel:** {name} | **Published:** {Mon DD} | **Duration:** {N} min

**Bold 1-sentence big picture.** What this video actually says.

- Key claim with evidence or example.
- Another specific claim, number, or story.
- Keep the same depth as the content — don't truncate.

---

{Repeat for each video. If none: "No new long-form content from tracked channels today."}

---

## Hacker News

### [{Article Title}]({url})

**Bold 1-sentence big picture.** What this article actually says.

- Concrete claim, fact, or insight.
- Specific number, name, or example.
- Every bullet earns its place.

---

{Only include relevant articles. Irrelevant ones are silently dropped.}

---

_Generated by Open Antenna · {date}_
```

### Writing standards

- **Plain language.** Write so it's understood at midnight without thinking.
- **No jargon shortcuts.** Unpack abbreviations and insider terminology inline.
- **Bullet points for lists.** Prose for context.
- **Titles are hyperlinks.** No separate URL lines.
- **No score/comment counts** in the digest output.
- **Only relevant items.** If it's not useful for this profile, it doesn't exist.
- **Dense but readable.** Every sentence earns its place and is immediately clear.

---

## Step 5: PDF Conversion (if configured)

If `output.format` is `pdf` in config:

```bash
node {lib_dir}/pdf.js {output_dir}/YYYY-MM-DD-digest.md
```

Output: `{output_dir}/YYYY-MM-DD-digest.pdf`

---

## Error Handling

| Failure | Action |
|---------|--------|
| YouTube scanner fails | Note in digest. Continue with HN. |
| HN scanner fails | Note in digest. Continue with YouTube. |
| Single video transcript unavailable | Skip. Note in digest footer: "{title} — transcript unavailable." |
| Single HN article inaccessible | Skip silently. |
| Both scanners fail | Write minimal digest noting failures. |
| PDF conversion fails | Log error. Markdown digest is still valid output. |

Never abort entirely. Partial briefing > no output.
